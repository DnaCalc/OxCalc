Why CTRO Exists

In an ordinary reactive system, the dependency graph is usually known before evaluation. If B1=A5, then changing A5 dirties B1. That part is simple, and OxCalc still treats it as the primary structural graph.

A spreadsheet has another problem: some grid values and some dependencies only become real while calculation is running. Dynamic arrays can publish values into cells that were structurally empty. INDIRECT and OFFSET can resolve a target from text or runtime arithmetic. Dynamic names can retarget while formulas are being evaluated. External and volatile formulas can need recalculation even when no authored formula changed.

Plain spill into ordinary reference

A1: 1
A2: =SEQUENCE(A1)
B1: =A5

B1 structurally depends on A5. When A1 changes to 5, A2 publishes a spill into A2:A6. Now A5 has an effective value, so ordinary structural invalidation must dirty B1.

Runtime-only target

C1: "A5"
B1: =INDIRECT(C1)

The authored formula does not contain a direct A5 reference. Evaluation discovers the realized target. That dependency belongs in the calc-time overlay graph and is replaced whenever the formula evaluates again.

The core correction

CTRO no longer means "predeclare every possible dynamic site." The live grid model is a layered graph plus publication deltas: 

effective calculation dependency graph
= structural/formula dependency graph
+ calc-time realized overlay graph

What changed from the old CTRO plan

The old grid grid/machine/rebind.rs claim lifecycle was deleted from the live grid machine. The old review terms DynamicRebindClaim, DynamicRebindConsequence, ResolvedReferenceIdentity, and resolve_dynamic_rebind_claim are historical context only. The current implementation uses runtime traces, overlay replacement, publication deltas, and effective-graph recalc.

The Model

1. Structural Graph

Authored formulas install dependencies that can be known before evaluation: direct cells, ranges, names, tables, spill-anchor references such as A2#, reference metadata, and axis dependencies.

2. Calc-Time Overlay

Evaluation records runtime-only realized dependencies such as INDIRECT, OFFSET, dynamic-name targets, table/name identity edges, volatility, and external pending state.

3. Publication Delta

Publishing a formula result tells the graph which effective cells changed or were vacated, which spill facts changed, and which spill blockers changed.

Authored structure Cells, formulas, names, tables, axes
Structural layer Dependencies known before evaluation
Calc overlay layer Runtime-only realized dependencies
Effective closure Dirty recalc sees both layers

The layers are deliberately separate. A direct formula such as B1=A5 is not rediscovered at runtime. It stays structural. A formula such as B1=INDIRECT(C1) installs a runtime overlay edge only after evaluation resolves the target.

Trace To Overlay Replacement

Source: src/oxcalc-core/src/grid/machine/runtime_trace.rs:203
pub fn replace_overlay_dependencies_from_trace(
    &mut self,
    dependent: ExcelGridCellAddress,
    trace: &GridRuntimeDependencyTrace,
) -> Result<GridOverlayDependencyUpdate, GridRefError> {
    let structural_dependencies =
        self.semantic_dependencies_for_layer(GridDependencyLayer::Structural, &dependent);
    let old_dependencies = self
        .semantic_dependencies_for_layer(GridDependencyLayer::CalcOverlay, &dependent)
        .into_iter()
        .collect::<BTreeSet<_>>();
    let mut overlay_dependencies =
        trace.overlay_dependencies_excluding_structural(&structural_dependencies);
    overlay_dependencies.extend(
        old_dependencies
            .iter()
            .filter(|dependency| matches!(dependency, GridDependency::SpillBlocker(_)))
            .cloned(),
    );
    self.set_volatile_root(dependent.clone(), trace.volatile)?;
    self.set_external_pending_root(dependent.clone(), trace.is_external_pending())?;
    if old_dependencies == overlay_dependencies {
        self.set_overlay_dependencies(dependent.clone(), overlay_dependencies)?;
        return Ok(GridOverlayDependencyUpdate::unchanged(
            dependent,
            old_dependencies,
        ));
    }
    self.set_overlay_dependencies(dependent.clone(), overlay_dependencies.clone())?;
    Ok(GridOverlayDependencyUpdate::changed(
        dependent,
        old_dependencies,
        overlay_dependencies,
    ))
}

Source: src/oxcalc-core/src/grid/machine.rs:153
#[must_use]
pub fn dirty_seeds(&self) -> BTreeSet<GridDirtySeed> {
    let mut seeds = BTreeSet::new();
    seeds.extend(
        self.changed_effective_cells
            .iter()
            .chain(self.vacated_effective_cells.iter())
            .cloned()
            .map(GridDirtySeed::Cell),
    );
    seeds.extend(
        self.changed_spill_fact_anchors
            .iter()
            .cloned()
            .map(GridSpillDependency::anchor)
            .map(GridDirtySeed::SpillFact),
    );
    seeds.extend(
        self.changed_spill_blocker_extents
            .iter()
            .cloned()
            .map(GridSpillBlockerDependency::extent)
            .map(GridDirtySeed::SpillBlocker),
    );
    seeds
}

Call Flows

These are the implementation patterns to keep in mind when reviewing or extending the current model.

Formula Install: Structural Dependencies

Parse and bind OxFml produces the bound formula and reference records.
Derive structural deps Cells, ranges, names, tables, spill anchors, metadata, axes.
Install structural layer set_structural_dependencies replaces authored deps.
Keep overlay separate Runtime discoveries are not mixed into authored edges.
Key source: runtime_trace.rs:275 and invalidation.rs:2274

Runtime Dynamic Reference: Trace To Overlay

Evaluate formula Provider resolves text and transforms.
Collect trace Realized deps, volatility, external state.
Subtract structural Direct authored deps are not duplicated.
Replace overlay Old realized deps are removed.
Seed retargets Added/removed deps become dirty seeds.
C1 changes from "A5" to "A7". Dirty closure reaches B1=INDIRECT(C1) through the structural dependency on C1.
B1 evaluates and resolves the new target. The trace contains a realized dependency on A7.
The overlay edge is replaced. A5 is removed, A7 is added, and future edits to A5 no longer dirty B1.
Key sources: runtime_trace.rs:203, calc_ref_sheet.rs:1220, optimized_sheet.rs:1805

Spill Growth Or Shrink: Publication Delta

Evaluate anchor Example: A2=SEQUENCE(A1).
Compare publication Old effective cells vs new effective cells.
Build delta Changed cells, vacated cells, spill facts, blockers.
Emit seeds Cell, SpillFact, SpillBlocker.
Close effective graph Ordinary refs and spill-anchor refs are dirtied.

This is the key grid correction. The engine does not need to know in advance that A5 might later be a spill cell. When a publication makes A5 changed or vacated, GridDirtySeed::Cell(A5) reaches structural and overlay consumers through the effective graph.

Key sources: machine.rs:100, calc_ref_sheet.rs:1236, optimized_sheet.rs:1832

Dirty Recalc Worklist

Receive seeds Edit, publication, name/table lifecycle, axis, volatile, external.
Close graph Structural plus calc overlay.
Pick ready formula Effective precedents, not raw address order.
Evaluate and trace Value plus runtime dependency trace.
Replace overlay and publish New dirty seeds may be emitted.
Requeue until stable Cycles are reported when discovered.
Key sources: calc_ref_sheet.rs:1177, optimized_sheet.rs:1745, invalidation.rs:2999

External And Volatile Roots

Trace marks root Volatile or external-pending formula/name.
Host reports event Broad availability or topic-specific dispatch.
Dirty seed closes Volatile or External reaches dependents.
Recalc refreshes state Successful evaluation clears or replaces subscriptions.
Key sources: machine.rs:297, machine.rs:352, runtime_trace.rs:73

Scenario Walkthroughs

Plain spill into ordinary reference

A2=SEQUENCE(A1), B1=A5. Growth from one row to five rows changes effective A5.

Mechanism: publication delta emits GridDirtySeed::Cell(A5); structural graph dirties B1.

Evidence: tests named grid_calc_ref_dirty_recalc_reaches_structural_dependent_when_spill_grows_into_cell and optimized_grid_dirty_recalc_reaches_structural_dependent_when_spill_grows_into_cell.

Spill anchor consumer

SUM(A2#) depends on the spill fact for anchor A2, not every currently occupied spill cell.

Mechanism: structural dependency is GridDependency::SpillFact; publication delta emits GridDirtySeed::SpillFact.

Evidence: tests named grid_calc_ref_dirty_recalc_reaches_spill_anchor_consumer_when_extent_changes and optimized counterpart.

Runtime scalar retarget

B1=INDIRECT(C1) retargets from A5 to A7.

Mechanism: C1 is structural; realized A5/A7 is overlay replacement state.

Evidence: tests named grid_calc_ref_dirty_recalc_replaces_indirect_overlay_on_retarget and optimized_grid_dirty_recalc_replaces_indirect_overlay_on_retarget.

Runtime dereference before precedent readiness

A1=MyFunc("A2"), A2=1+1, where MyFunc evaluates the text reference A2 internally.

Excel observation: a local VBA probe on Excel 16.0 (build 20026) saw MyFunc called once with A2.Value2 / Evaluate("A2") as <Empty>, then called again after A2 evaluated to 2.

Mechanism implication: the UDF invocation can happen before the target is ready; correctness comes from the caller being rerun after the discovered precedent publishes.

Boundary: the probe does not prove whether Excel internally commits, suppresses, or later overwrites the first result.

Runtime range retarget

SUM(INDIRECT(C1)) retargets from one range to another.

Mechanism: the calc-overlay layer stores the realized GridDependency::Range; interior cells of the current range dirty the formula.

Evidence: tests named grid_calc_ref_dirty_recalc_replaces_indirect_overlay_range_and_dirties_from_inside and optimized counterpart.

OFFSET retarget

OFFSET(A1,N1,0) keeps base and offset inputs structural while the realized target moves.

Mechanism: runtime reference transform records the realized target in the overlay layer and replaces it after reevaluation.

Evidence: tests named grid_calc_ref_dirty_recalc_replaces_offset_overlay_on_retarget and optimized counterpart.

Branch flip through IF

A formula switches from INDIRECT("A5") to INDIRECT("A7").

Mechanism: the overlay dependency set is replacement state, so the inactive branch's old realized target is released.

Evidence: tests named grid_calc_ref_dirty_recalc_replaces_overlay_when_if_branch_flips and optimized counterpart.

Name deletion vs empty target

Clearing the target value of a name is not the same as deleting the name from the namespace.

Mechanism: value changes flow through the name's realized extent; namespace deletion flows through GridDirtySeed::Name and release semantics.

Evidence: tests named grid_calc_ref_dirty_recalc_distinguishes_empty_name_target_from_name_delete and optimized counterpart.

Dynamic defined-name retarget

A dynamic name such as InputRange=INDIRECT(C1) changes realized extent.

Mechanism: the dynamic-name formula has its own input ledger; extent changes emit ordinary name dirty seeds for consumers.

Evidence: tests named grid_calc_ref_dirty_recalc_retargets_dynamic_defined_name_for_direct_consumer and optimized counterpart.

Table lifecycle

Structured references must respond to table resize, rename, delete, and recreate.

Mechanism: table lifecycle reports produce GridDirtySeed::Table; missing table identity dependencies can heal on later creation.

Evidence: tests named grid_calc_ref_dirty_recalc_table_rename_delete_and_recreate_reaches_structured_consumers and optimized counterpart.

Table/spill blocker interaction

A table can grow into a dynamic-array spill extent, or a spill can grow into a table extent.

Mechanism: publication-owned spill-blocker watches live in the overlay layer; lifecycle reports emit paired table and blocker dirty seeds.

Evidence: tests named grid_calc_ref_dirty_recalc_table_resize_blocks_unrelated_spill_anchor and grid_calc_ref_dirty_recalc_spill_growth_into_table_blocks_and_releases_anchor_consumer.

Volatile root and warm-no-op safety

Volatile formulas and volatile dynamic names must recalc even when their apparent target is stable.

Mechanism: traces mark volatile roots; GridDirtySeed::Volatile reaches dependents; optimized warm-no-op refuses unsafe reuse.

Evidence: tests named optimized_grid_volatile_dynamic_defined_name_blocks_warm_noop_and_dirties_consumer and dirty-recalc volatile root tests.

External pending/topic availability

External data may become available without an authored formula edit.

Mechanism: runtime traces record pending roots and topic subscriptions; host reports dirty exact topic roots or broad availability roots.

Evidence: tests named grid_external_topic_registry_dispatches_trace_produced_subscription, grid_calc_ref_external_topic_event_dirties_only_subscribed_pending_formula_root, and optimized counterparts.

Axis edit re-realization

After inserting a row, direct references transform, but textual references such as INDIRECT("A5") should re-resolve as text.

Mechanism: axis edit transforms the structural layer and clears overlay realization state; evaluation rebuilds overlay dependencies.

Evidence: tests named grid_calc_ref_axis_edit_clears_overlay_and_rerealizes_indirect_text and optimized valuation counterpart.

Runtime/effective cycle detection

A dynamic reference can introduce a cycle only after runtime resolution.

Mechanism: dirty recalc uses the effective graph for readiness and reports effective dependency cycles when runtime overlay edges create one.

Evidence: tests named grid_calc_ref_mark_all_reports_dynamic_self_cycle_after_resolution and stale-overlay cycle release tests.

Why We Believe It Is Correct

Invariant Implementation evidence Why it matters
Direct references remain structural-only. GridRuntimeDependencyTrace::overlay_dependencies_excluding_structural filters realized dependencies covered by structural dependencies. Runtime does not duplicate known edges such as B1=A5.
Runtime discoveries are replacement state. replace_overlay_dependencies_from_trace computes old/new overlay sets and installs the new set. Retargeting does not accumulate stale dependencies.
Removed overlay targets stop dirtying formulas. GridOverlayDependencyUpdate records added/removed dependencies and tests assert old targets no longer reach the formula. Incremental recalc stays precise enough to trust.
Spill changed/vacated cells are ordinary dirty seeds. GridValuePublicationDelta::dirty_seeds() emits GridDirtySeed::Cell for changed and vacated effective cells. Plain refs and ranges update when spill cells appear or disappear.
Effective dirty closure walks both graph layers. GridInvalidationRef owns structural and calc_overlay; dirty closure APIs close over the effective graph. Structural and runtime dependents are reached by the same recalc input.
Order comes from dependency readiness. Reference and optimized recalc use next_ready_dirty_formula before evaluation. Forward formula chains do not read stale precedents merely because of address order.
Dirty recalc is checked against mark-all. GridDirtyRecalcDifferentialRunReport compares values, spill facts, graph state, and dynamic-name state. The incremental path has a practical oracle for new scenario families.

Current verification gate

The parking checkpoint records cargo test -p oxcalc-core --lib as green with 835 passed and 2 ignored. This presentation should stay aligned with that live code and should be rechecked whenever the architecture or code fragments change.

Excel observation boundary

The VBA UDF probe supports the effective-graph requeue shape: a formula can run early, discover a runtime dependency, observe an empty or stale value, and then run again after the discovered precedent publishes. It does not prove Excel's private commit/reject mechanics for the first invocation. A future scheduler refinement may still want an explicit provisional-result state for evaluations that touched not-yet-ready formula precedents.

Covered Scope And Remaining Refinements

Covered by the current implementation

  • Spill growth and shrink into ordinary cells and ranges.
  • Spill-anchor dependencies such as A2#.
  • INDIRECT, OFFSET, runtime scalar targets, and runtime ranges.
  • Branch-sensitive overlay replacement.
  • Static, scoped, and dynamic defined names.
  • Structured table resize, rename, delete, and recreate.
  • Table/spill blocker interactions.
  • Axis value and visibility dependencies.
  • Volatile roots, external pending roots, topic registry dispatch.
  • Runtime/effective cycle reporting.
  • Dirty-vs-mark-all differential verification.

Still deliberate refinements

  • Exact external provider/workspace topic descriptors.
  • Cross-sheet and 3D references.
  • Rich objects and FIELDVALUE.
  • Excel-style iterative calculation.
  • Provisional publication semantics for UDF/dynamic dereference calls that observe not-yet-ready formula precedents.
  • Legacy CSE arrays.
  • Exact Excel-version switches for version-sensitive behavior.
  • Broader performance hardening before scale claims.
  • Public API polish after real host integration use.

How To Put This Model Into Use

  1. Install structural dependencies when formulas are created, rebound, or structurally transformed.
  2. Run dirty recalc from report-produced dirty seeds for edits, name/table lifecycle, axis edits, publication deltas, volatility, and external availability.
  3. Treat overlay dependencies as evaluation-owned replacement state. Do not append new runtime targets to old ones.
  4. Apply GridExternalAvailabilitySubscriptionUpdate records to the host-owned topic registry after recalc.
  5. For broad external availability changes, use GridExternalAvailabilityEventReport. For recognized topics, use topic dispatch.
  6. For new families, run dirty-vs-mark-all differential checks before relying on incremental results.
