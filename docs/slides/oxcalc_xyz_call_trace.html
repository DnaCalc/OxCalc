The exercised path now evaluates direct TreeCalc references through OxFml/OxFunc, builds the dependency graph in OxCalc, and publishes the recalculated values through the Stage 1 coordinator.

Call Structure

OxCalc owns graph, invalidation, scheduling, and publication

1. Load tree substrate

StructuralSnapshot carries nodes X, Y, Z. TreeFormulaCatalog carries formulas for Y and Z.

2. Prepare formulas

prepare_oxfml_formula translates TreeCalc formulas into OxFml formula text and synthetic reference names.

3. Build graph

DependencyGraph::build lowers bind facts into descriptors, forward edges, reverse edges, and diagnostics.

4. Evaluate locally

evaluate_via_oxfml builds a minimal upstream host packet and calls OxFml with OxFuncBacked.

5. Publish

TreeCalcCoordinator admits the accepted candidate and publishes one atomic value delta.

Main Phases

The current evaluation path has five important phases

Prepare

Tree formulas are rendered as formula entries, parsed by OxFml, bound with caller and structure context, and mapped to synthetic reference carriers.

Derive dependencies

OxCalc lowers bound references into dependency descriptors, forward edges, reverse edges, cycle groups, and replay-visible diagnostics.

Close invalidation

Seeds flow through reverse edges. In this example, X invalidates Y and then Z; rebind is not required for direct references.

Evaluate candidates

OxCalc creates a minimal upstream host packet. OxFml executes the formula with OxFunc-backed semantics and returns candidate/commit information.

Publish or reject

Accepted candidates become one coordinator publication bundle. Rejects publish no derived value delta and remain diagnostic/replay-visible.

Scope Boundary

What this proves, and what it does not claim

Supported here

  • Tree-only local sequential recalculation.
  • Direct node dependencies with arithmetic formulas.
  • OxFml/OxFunc-backed formula evaluation via the minimal host packet.
  • Explicit dependency graph, invalidation closure, candidate adaptation, and coordinator publication.

Not claimed by this slide

  • Full grid/workbook semantics.
  • All relative, dynamic, host-sensitive, shape, topology, format, or display families.
  • Stage 2 parallel publication or broad scheduler policy promotion.
  • Full cross-repo seam closure beyond the exercised direct-reference slice.
