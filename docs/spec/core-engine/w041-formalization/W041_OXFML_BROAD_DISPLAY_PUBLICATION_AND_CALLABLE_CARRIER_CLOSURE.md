## OxFml Formatting Intake

The latest OxFml formatting update is incorporated as a consumed boundary, not as a new implementation dependency or handoff trigger.

Current W041.8 consequence:

1. W073 remains a direct-replacement input contract for aggregate and visualization conditional-formatting metadata.
2. `VerificationConditionalFormattingRule.typed_rule` remains the only accepted metadata source for `colorScale`, `dataBar`, `iconSet`, `top`, `bottom`, `aboveAverage`, and `belowAverage`.
3. W072 bounded `thresholds` strings are not a fallback for those aggregate and visualization families.
4. OxFml now explicitly keeps `format_delta` and `display_delta` as distinct canonical bundle categories.
5. OxFml currently describes the semantic-format/display-facing boundary as canonical but narrower, and says the note does not require a formal handoff by itself.
6. OxCalc therefore retains the typed-only formatting guard and the distinct format/display rows while keeping broad display/publication unpromoted.

No OxFml repo files were edited by this packet.
