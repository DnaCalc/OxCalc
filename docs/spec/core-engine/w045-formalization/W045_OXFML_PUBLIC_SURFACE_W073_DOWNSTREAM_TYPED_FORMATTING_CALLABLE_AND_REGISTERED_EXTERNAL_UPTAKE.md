## 2. OxFml Formatting Intake

OxFml was inspected read-only before this packet. The local formatting update is incorporated as evidence:

1. `VerificationConditionalFormattingRule.typed_rule` is the only accepted metadata source for `colorScale`, `dataBar`, `iconSet`, `top`, `bottom`, `aboveAverage`, and `belowAverage`.
2. W072 bounded `thresholds` strings are intentionally ignored for those aggregate and visualization families.
3. `thresholds` remains meaningful for scalar/operator/expression rule families where threshold text is the real rule input.
4. OxFml has focused old-string non-interpretation tests for visualization and aggregate option strings.
5. OxCalc fixture-side W073 request construction emits `typed_rule` for the exercised rank, average, color-scale, data-bar, and icon-set cases.
6. DNA OneCalc downstream typed-rule request construction remains required and unverified by OxCalc.

No OxFml files are edited by this packet.
